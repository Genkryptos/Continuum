Memory for AI agents is a
state problem, not a retrieval problem.

Continuum is a memory layer with first-class supersession and bi-temporal recall — it knows which version of a fact is current, and what was true as of any date.

The numbers, honestly

100%scripted supersession & as-of-date correctness
~5ktokens per query (Mem0-competitive)

LongMemEval-S is ~74% (73.6–75.6% across repeated runs; gpt-oss-120b has ±3–5pp variance). The read-side additions are within that noise of the 73.8% control — the reproducible wins are the 100% deterministic benches. See the report.

Use it in three lines

pip install continuum-memory

from continuum import Memory
mem = Memory.in_memory()
await mem.add("I moved to Boston")
await mem.add("Actually I'm in NYC now")   # supersession handles the update
await mem.current("user", "residence")     # -> "NYC" (resolved, not stale)
await mem.timeline("residence")            # bi-temporal history

Or plug it into any MCP client (Claude Code, Cursor):

pip install "continuum-memory[mcp]"
continuum-mcp        # exposes recall / remember / current / timeline as tools

What we tried that didn't work

We spent real effort proving four ideas wrong — counting in code, a deterministic answer router, evidence distillation, temporal date-math — all net-negative, because you can't bolt deterministic machinery onto a mid-size model's unreliable intermediate outputs. The prompt/sampling levers we kept are about accuracy-neutral (within noise) — the reproducible wins are the deterministic supersession benches above. The negative results, and the measurement mistake that made them look good at first, are in the report.
